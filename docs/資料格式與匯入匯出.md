# 資料格式與匯入／匯出說明

負荷漸進追蹤（fit-track）是 **local-first** App：所有資料存在使用者裝置瀏覽器的 **IndexedDB**，沒有伺服器、沒有帳號。備份與搬移靠「設定」頁的匯出／匯入。

---

## 1. 資料模型（IndexedDB）

共四個資料表（object stores）：

| 表 | 欄位 | 說明 |
|---|---|---|
| `exercises` 動作 | `id`, `name`, `muscles: [{muscle, weight}]`, `createdAt` | `muscles` 為肌群陣列，`muscle` ∈（胸／背／肩／二頭／三頭／腿臀），`weight` 目前固定 1 |
| `templates` 範本 | `id`, `name`, `exerciseIds: [id...]`, `createdAt` | 一套動作清單，順序即套用時帶出的順序 |
| `sessions` 訓練 | `id`, `date`（`YYYY-MM-DD`）, `templateId\|null`, `note`, `committed`(bool), `exerciseIds: [id...]`, `createdAt` | `committed=false` 為尚未按「完成」的草稿；`note` 常存課表／分部名 |
| `sets` 組 | `id`, `sessionId`, `exerciseId`, `setOrder`, `weight`(kg), `reps`, `isWarmup`(bool), `done`(bool) | 衍生值（容量＝weight×reps 等）查詢時即算、不存 |

- 報表只計算 `isWarmup=false` 的「正式組」。
- `done=false` 的組在「完成」時會被丟棄（只有打勾的組進入紀錄）。

---

## 2. JSON 匯出／匯入（完整備份）

JSON 是**完整備份**，包含上述四個表的全部內容（動作庫、範本、訓練、組）。

### 檔案格式
```json
{
  "app": "fit-track",
  "version": 1,
  "exportedAt": "2026-06-14T12:00:00.000Z",
  "data": {
    "exercises": [ { "id": "...", "name": "臥推（槓鈴）", "muscles": [{ "muscle": "胸", "weight": 1 }], "createdAt": 1700000000000 } ],
    "templates": [ { "id": "...", "name": "範本（推日）", "exerciseIds": ["...", "..."], "createdAt": 0 } ],
    "sessions": [ { "id": "...", "date": "2026-06-13", "templateId": null, "note": "周六（胸+肩+三頭）", "committed": true, "exerciseIds": ["..."], "createdAt": 0 } ],
    "sets": [ { "id": "...", "sessionId": "...", "exerciseId": "...", "setOrder": 0, "weight": 30, "reps": 10, "isWarmup": false, "done": true } ]
  }
}
```

### 機制
- **匯出 JSON 備份**：把整個資料庫倒成上面的檔案下載。
- **匯入 JSON 還原（覆蓋）**：先**清空**四個表，再寫入檔案內容 → 完全變成檔案的狀態（適合換機、誤清救援）。
- **合併匯入 JSON（加進現有）**：以 `id` 為鍵**逐筆寫入、不清空** → 既有資料保留，檔案內的項目新增或覆蓋同 `id` 者（適合把另一份紀錄／動作併進來）。

> 要保留「動作庫的肌群分類、範本、草稿」等完整資訊，請用 JSON。

## 3. CSV 匯出／匯入（訓練紀錄）

CSV 是**訓練紀錄表**，方便用試算表檢視或與其他 App 交換。**每一列＝一組**。

### 欄位（App 匯出格式）
```
日期,課表,動作,組,重量,次數,熱身,完成
2026-06-13,周六（胸+肩+三頭）,臥推（槓鈴）,1,30,10,0,1
```
| 欄 | 說明 |
|---|---|
| 日期 | `YYYY-MM-DD` |
| 課表 | 範本名或該次的分部名（`note`），可空白 |
| 動作 | 動作名稱（匯入時用名稱比對動作庫）|
| 組 | 該動作第幾組（匯入時僅參考，會重新編號）|
| 重量 | kg |
| 次數 | reps（**必填**；空白或 0 的列會被略過）|
| 熱身 | 1＝熱身組、0＝正式組 |
| 完成 | 1＝已完成、0＝未完成 |

### 機制
- **匯出 CSV**：把所有訓練的每一組輸出成上表（含 UTF-8 BOM，Excel／Google 試算表開中文不亂碼）。
- **匯入 CSV**：
  1. 解析並判斷格式。支援兩種表頭：
     - **本 App 格式**：`日期, 課表, 動作, 組, 重量, 次數, 熱身, 完成`
     - **FitFit／Strong 格式**：`Date, Workout Name, Exercise Name, Set Order, Weight, Reps`（其餘欄忽略；熱身預設 0、完成預設 1）
     - 只要含「日期/Date、動作/Exercise Name、次數/Reps」即可解析；都對不到 → 提示「此 CSV 無法使用」。
  2. 動作名稱會**正規化**（半形括號→全形）後，用名稱比對現有動作庫：對得上→沿用（保留肌群）；對不到→**新建一個空動作（無肌群）**。
  3. 依（日期＋課表）分組成一次訓練（`committed=true`，`note`＝課表）。
  4. 套用方式：**清空並覆蓋 `sessions` 與 `sets`**、**保留現有 `templates`**、**合併 `exercises`（只新增缺少的）**。

### CSV 的限制（重要）
- **不含動作庫的肌群分類**：CSV 只有動作「名稱」。匯入後對不到既有動作庫的動作會是「無肌群」，需到動作庫補分類（或改用 JSON）。
- **不含範本**：CSV 沒有範本資訊（但匯入時會保留裝置上現有的範本）。
- **不含空白訓練**：沒有任何組的訓練無法用 CSV 表示；同一天同課表會被合併成一次。
- **匯入會覆蓋訓練紀錄**：CSV 匯入是「以這份取代全部訓練紀錄」，不是新增。要新增請用 JSON 的「合併匯入」。

---

## 4. 什麼時候用哪個？

| 目的 | 用 |
|---|---|
| 完整備份／換機／誤清救援 | **JSON 匯出 / 匯入還原** |
| 把另一份紀錄或動作併進來、不想動到現有 | **JSON 合併匯入** |
| 用試算表看、編輯、或跟別的 App 交換紀錄 | **CSV 匯出 / 匯入** |

> 慣例提醒：本資料以「雙手總重」記錄。啞鈴等單邊動作若原始只記單手重量，請自行 ×2 後再填／匯入。
